# Consistency Checklist

Use this before publishing the website, sending the CV, or submitting job-market materials.

## Identity and Contact

- [ ] Name is identical across `source/profile.yml`, CV, bios, website pages, and job-market materials.
- [ ] Title, department, institution, and location match current status.
- [ ] Email, website, Google Scholar, LinkedIn, GitHub, and ORCID links are current.
- [ ] The same pronouns or sentence style are used consistently across bios.

## Research

- [ ] Research interests use the same core language across CV, bio, website, and statements.
- [ ] Project titles match across `source/research.yml`, `source/publications.yml`, CV, and website.
- [ ] Coauthor names are spelled consistently.
- [ ] Paper statuses are current and not overstated.
- [ ] Public website language does not disclose confidential review, data, or partner information.

## Publications and Presentations

- [ ] Citation style is consistent.
- [ ] Publication years, venue names, DOI links, and author order are correct.
- [ ] Working papers and work in progress are separated clearly.
- [ ] Talks and presentations match conference programs or official titles.

## Teaching and Service

- [ ] Course titles, roles, terms, and institutions are accurate.
- [ ] Teaching evaluations or enrollment numbers are included only when verified.
- [ ] Service roles are not double-counted across categories.

## Versioning

- [ ] Job-market CV is tailored but factually consistent with the main CV.
- [ ] Website pages are public-facing and shorter than application documents.
- [ ] `source/last_updated` and visible CV date are refreshed before distribution.
